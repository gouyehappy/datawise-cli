# 客户端对标优化 backlog（Chat2DB / DBeaver / Navicat / DataGrip / TablePlus）

> 记录可调整点：每条必须写明 **对标其它 IDE 的优势**，以及 **对本系统（DataWise）的帮助**。  
> 定位原则：补齐「日常 DBA/开发手感」，保持「AI + 团队治理 + 跨源」差异化，不做又一个万能 JDBC 壳。

状态：`todo` 未开始 · `partial` 已有 MVP · `done` 已闭环  
优先级：`P0` 高收益日常路径 · `P1` 中期体验 · `P2` 按需补齐

对照产品：

| 产品 | 典型优势 |
|------|----------|
| **DataGrip** | Schema 感知补全、写时校验、跳转/Find Usages、安全重构 |
| **DBeaver** | 多库、ER 成熟、数据传输向导、可视化 EXPLAIN、Visual Query Builder（Pro） |
| **Navicat** | 结构/数据同步、可视化建模、备份还原、Form 编辑 |
| **Chat2DB** | Text-to-SQL 易用、结果一键出图、分析向 BI 轻量闭环 |
| **TablePlus** | 轻量、连接切换快、键盘优先、网格手感干净 |

DataWise 已具备、应保持的差异化（不必用竞品同款替代）：AI 分析画布、语义指标、联邦视图、危险 SQL/生产审批、Schema 漂移、MCP/VS Code、脱敏导出、Mongo/Redis/Kafka 工作台、网格 Time-travel 审计等。

## P0 — 优先落地

### 1. SQL IDE 智能深度（别名 / CTE / 写时校验 / 跳转）

| 项 | 内容 |
|----|------|
| **状态** | `done`（别名/CTE 补全已有；新增 `alias.column` 写时校验 Monaco markers） |
| **调整点** | 补齐别名与 CTE/子查询上下文补全；未声明列/歧义列写时标红；表/列/视图跳转到定义与 Find Usages；重命名列等安全重构预览 |
| **对标其它 IDE 优势** | **DataGrip** 在复杂 SQL（多 JOIN、CTE、子查询）上的 schema 感知补全与导航明显领先 DBeaver/Navicat；是开发者长期留存的核心。**DBeaver** 基础补全可用但深度不足。 |
| **对本系统的帮助** | 降低「会 AI 但手写复杂 SQL 仍痛苦」的落差；与现有 `@datawise/sql-editor`、控制台、危险 SQL 审查形成「写得准 → 跑得稳 → 审得过」闭环；提升相对 Chat2DB「只会聊天生成」的专业可信度。 |
| **主要落点** | `sql-editor/`、控制台 Tab、Schema 元数据索引 |

### 2. 结构同步 / 数据同步向导（Compare → 勾选 → 执行）

| 项 | 内容 |
|----|------|
| **状态** | `done`（结构：对比 → 勾选 → 控制台打开 / 执行结构同步 / 生产审批；数据：迁移向导预填为 `partial`） |
| **调整点** | 结构差异勾选后一键生成并执行同步脚本；按主键/唯一键的数据增量同步（冲突策略、进度、可中断）；dev→staging 等环境流水线模板 |
| **对标其它 IDE 优势** | **Navicat** 的 Structure Sync / Data Sync 是迁移与发版刚需，完成度高于多数免费客户端。**DBeaver Pro** 有部分对比能力，深度不及 Navicat。Chat2DB/TablePlus 在此较弱。 |
| **对本系统的帮助** | 把现有「对比报告」升级为「可执行同步」，打通治理链路（环境标签、生产审批、危险 SQL）；强化跨环境对比已有能力的商业价值；减少用户为同步回流 Navicat。 |
| **主要落点** | `schema-compare`、迁移向导、平台漂移报告 |

### 3. 查询结果轻量可视化（一键图表 + 简单透视）

| 项 | 内容 |
|----|------|
| **状态** | `done`（表格|图表切换 + 维度/度量/透视 + **保存为 Dashboard 图表 Widget**） |
| **调整点** | 结果网格一键柱状/折线/饼图（自动识别维度/度量）；简单 Pivot；「保存为 Dashboard Widget」一键落位 |
| **对标其它 IDE 优势** | **Chat2DB** 强在「查完立刻出图」，分析师路径极短。传统 DBeaver/Navicat 也有图表，但 AI+图表一体化是 Chat2DB 卖点。 |
| **对本系统的帮助** | 补齐日常「跑查询 → 看趋势」无需进入完整 AI 流水线；与自然语言 Dashboard、语义指标复用；缩短 Onboarding「30 秒出洞察」路径。 |
| **主要落点** | `QueryResultPane`、Dashboard、分析导出 |

## 明确不做或慎做（避免稀释定位）

| 项 | 原因 |
|----|------|
| 再做一个「大而全 JDBC 万能壳」 | 已有连接器 SPI + 插件市场路径，重复建设 ROI 低 |
| 用 AI 完全替代 SQL 编辑智能 | 应 DataGrip 级智能 + AI 加速双轨 |
| 全面做成 Metabase 级 BI | 轻量出图 + 语义指标即可，避免产品重心漂移 |
| 空间/GIS 等小众类型 | 按客户需求再开，不进默认 P0/P1 |
