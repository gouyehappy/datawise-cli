# 04 · SQL 控制台

本章说明如何打开控制台、编写与执行 SQL、使用结果网格、书签、可视化查询构建与控制台内 AI。

---

## 4.1 功能说明

SQL 控制台是日常查询与治理的核心：Monaco 编辑、多语句执行、结果网格、事务、执行计划、书签，以及可视化查询构建（VQB）与 Text-to-SQL。

![SQL 控制台](../assets/screenshots/03-sql-console.png)

**图中信息：**

| 区域 | 说明 |
|------|------|
| Tab 栏 | 可同时开多个控制台 / 表 / 平台页 |
| 工具栏 | 执行选中、保存、格式化、可视化查询、AI 等 |
| 上下文条 | 数据源 / 实例 / 模式（务必先选对） |
| 编辑器 | Monaco，支持高亮、补全、多语句 |
| 结果区 | 网格列头与单元格；示例已跑通 `SELECT 1` → `id = 1` |

---

## 4.2 如何打开控制台

任选一种方式：

| 方式 | 步骤 |
|------|------|
| 仪表盘 | 「新建 SQL 控制台」或快捷操作 |
| 顶栏 / 工作台 | 新建控制台入口 |
| 资源树 | 数据库节点右键 → **SQL 编辑器** / **SQL 控制台** / **新建 SQL 编辑器** |
| 资源树 | **双击数据库节点**（会直接打开编辑器，上下文绑到该库） |
| 表节点 | 右键 → **查询控制台** |
| AI 分析 | 结果卡片 → **在控制台打开** |
| 命令面板 | `Ctrl+K` 搜「SQL」 |

打开后先看顶部 **数据源 / 数据库** 是否正确，错误上下文是最常见的「查不到表」原因。

## 4.3 编写与执行（逐步）

### 基本流程

1. 确认上下文：连接 + 库（+ Schema，若有）。
2. 在编辑器输入 SQL；多语句用分号分隔。
3. **只跑一段时**：用鼠标选中该段 → 点 **执行选中**（不要整文件误跑）。
4. 成功：下方出现结果网格；失败：阅读错误面板（语法、权限、超时）。

### 工具栏常用项

| 操作 | 作用 |
|------|------|
| 执行选中 | 运行当前选区；无选区时行为以产品默认为准（常为当前语句/全文） |
| 格式化 SQL | 美化缩进 |
| 保存 / 另存为文件 | 落盘到工作区脚本 |
| 保存书签 | 进入 Query Library，便于复用 |
| 保存为视图模型 | 固化为可打开数据的视图模型 |
| 可视化查询构建 | 打开 VQB 向导（见 4.6） |
| AI / Text-to-SQL | 自然语言生成 SQL（见 4.7） |

### EXPLAIN / 执行计划

- 使用产品提供的 Explain 入口（或手写 `EXPLAIN`）。
- 注意 **ANALYZE** 类选项会真实执行语句，生产库慎用。

### 事务

1. 开启事务（工具栏或数据源能力菜单）。
2. 执行多条 DML。
3. **提交** 或 **回滚**。
4. 不支持事务的引擎会提示或按钮不可用。

### 危险 SQL 与审批

对 `DROP` / `TRUNCATE` / 大批量 `UPDATE`/`DELETE` 等：

- 可能弹出确认、影响行预览或 AI 改写建议；
- 生产环境标签下工具栏会出现 **提交审批**（不要直接执行）；管理员在团队里 **批准并执行**（详见 [第 10.6 节](./10-team-governance.md#106-生产-sql-审批写操作)）；
- 访客账号通常无法执行写操作。

## 4.4 结果网格怎么用

| 能力 | 怎么做 |
|------|--------|
| 浏览 | 滚动、分页或窗口加载（视结果大小） |
| 排序 / 筛选 | 点列头或过滤控件（连接器支持时） |
| 单元格编辑 | 可写源 + 有权限时直接改，再保存 |
| Time-travel | 查看本地网格变更审计，一键回滚（非数据库闪回） |
| 导出 CSV | 插件启用时，从结果区导出；注意脱敏策略 |

大结果集建议在 SQL 中加 `LIMIT`，避免一次拉爆内存。

## 4.5 保存、书签与脚本文件

### 保存书签

1. 写好 SQL → **保存书签**。
2. 之后用命令面板或书签列表打开。
3. 适合周报查询、巡检语句。

### SQL 文件

- 资源树中可 **打开 SQL 文件** / **运行 SQL 文件**。
- 右键文件可 **定时执行…**、**版本历史**、重命名、删除。
- 适合纳入版本管理的脚本。

### 视图模型

**保存为视图模型** 后，可在树中打开数据、看血缘、再编辑 SQL（见资源树视图模型菜单）。

## 4.6 可视化查询构建（VQB）

适合不会或不想手写 JOIN 的场景。

### 入口

控制台工具栏 → **可视化查询构建**。

### 四步向导

| 步骤 | 做什么 | 技巧 |
|------|--------|------|
| 1 选表 | 选主表；可搜索表名 | 单击选中，双击可进入下一步 |
| 2 选字段 | 勾选或拖到排序板 | 都不选 = `SELECT *` |
| 3 关联 | 画布拖表或列表配置 JOIN | 单表可跳过 |
| 4 条件 | 筛选、排序、条数（可选） | 右侧实时预览 SQL |

### 收尾

1. 确认右侧 SQL 无误。
2. 点 **替换编辑器内容**（或同等按钮）写回 Monaco。
3. 再点 **执行选中** 跑结果。

未选数据源/库时会提示「请先选择数据源与数据库」。

---

## 4.7 控制台内 Text-to-SQL

1. 确保 **设置 → AI 模型** 已配置（截图 `09-settings-ai.png`）。
2. 在控制台 AI 输入区用自然语言描述需求（快捷键常为 `Alt+/` 唤起，以界面为准）。
3. 生成 SQL 后审阅，再执行。
4. 复杂分析更推荐完整 **AI 分析工作台**（第 6 章）：可选范围、流水线、图表与画布。

---

## 4.8 推荐工作流（日常）

```text
选对库上下文 →（可选）VQB 或 AI 生成草稿 → 人工审阅 SQL
    → 执行选中 → 看网格 → 导出 / 存书签 / 存画布
```

生产写操作：先在开发连接演练 → 备份（第 5 章）→ 再迁到生产并走审批。

---

## 4.9 常见问题

| 问题 | 处理 |
|------|------|
| 表不存在 | 上下文选错库；大小写/Schema；刷新树 |
| 执行按钮灰 | 未连接；访客只读；会话断开 |
| 结果为空但无报错 | 条件过严；权限导致空集；看是否跑错语句 |
| AI 生成不可用 | 配置 API Key；检查网络与配额 |
| CSV 导出没有 | 启用导出相关插件；检查权限 |
