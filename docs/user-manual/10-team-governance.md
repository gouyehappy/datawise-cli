# 10 · 团队与治理

面向团队管理员与 co-admin：共享连接、环境隔离、危险 SQL、生产审批、权限、租户、审计与通知。

---

## 10.1 你要先想清楚的策略

| 问题 | 建议落点 |
|------|----------|
| 谁能连生产？ | 环境标签 + 连接共享级别 + 账号权限 |
| 谁能跑 DROP？ | 危险 SQL 权限 + 生产审批 |
| AI 能看哪些表？ | AI 打标 + 数据范围（第 5–6 章） |
| 出了事如何追溯？ | 审计导出 + 通知通道 |

---

## 10.2 团队与共享连接

### 步骤

1. 创建或加入团队（团队入口 / 设置中的组织相关页）。
2. 所有者将连接共享给团队，并选择权限级别，例如：
   - **只读**：查数、导出（受策略限制）
   - **读写**：可 DML
   - **DDL**：可改结构
3. 成员刷新资源树，看到带「团队共享」标记的连接。
4. 无管理权的成员 **不能** 擅自改密码或删除共享连接。

### 注意

- 旧连接若无所有者，部分 DDL（如建库）可能被拒绝——让成员使用自己创建或明确授权的连接。
- 仪表盘「值班连接包」等组件在入队后更有意义。

---

## 10.3 环境标签

在 **新建/编辑连接** 表单中设置：

| 标签 | 建议用途 | 产品行为倾向 |
|------|----------|--------------|
| 开发 | 日常开发、测试数据 | 较少拦截 |
| 预发 | 发布前验证、漂移对比 | 中等确认 |
| 生产 | 真实业务 | 强化备份/还原确认、危险 SQL、审批 |
| 自定义 | 如 UAT | 按团队约定 |

**操作建议：** 生产连接名称也带 `prod` 前缀，避免看错树节点。

## 10.4 危险 SQL 审查（执行前）

当用户执行高风险语句时，系统可能：

1. 用客户端规则拦截（例如无 `WHERE` 的 `UPDATE`/`DELETE`）。
2. 结合执行计划深度提示全表扫描等风险。
3. 提供 **AI 改写** 建议 → 用户确认后写回控制台。
4. 无「危险 SQL」相关权限的账号无法绕过。

管理员应在 **账号权限** 中明确谁能绕过/执行危险类操作。

## 10.5 加入团队审批（邀请码）

### 管理员侧

1. 打开 **团队** 页，选中团队。
2. **共享设置 / 团队设置** 中开启 **加入团队需管理员审批**。
3. 把 **邀请码** 复制发给同事。
4. 同事提交后，到 **审批** Tab（或通知里的「前往审批」）处理：
   - **批准** → 对方成为成员
   - **拒绝** → 对方无法加入

### 成员侧

1. **加入团队** → 输入邀请码。
2. 若开启了审批：提示「加入申请已提交，等待管理员审批」。
3. 在「我的团队」可见「审批中」状态，通过后再 **进入工作台**。

## 10.6 生产 SQL 审批（写操作）

生产环境标签的连接上，写操作通常 **不能直接执行**，而要走「提交审批 → 管理员批准并执行」。

### 成员：提交审批

1. 确认当前控制台连的是 **生产** 标签连接。
2. 写好 `UPDATE` / `DELETE` / DDL 等语句。
3. 若出现提示：请使用工具栏 **提交审批**，不要点普通执行。
4. 点 **提交审批**，在对话框确认：
   - 团队
   - 连接
   - SQL 全文
5. 点对话框内 **提交审批** → 提示「已提交审批」。
6. 等待管理员处理；可在通知抽屉查看进度类事件（若已订阅 `prod_approval_*`）。

危险 SQL 审查（无 WHERE 等）仍可能先弹出确认/改写；生产环境无法靠白名单跳过确认。

### 管理员：批准并执行 / 驳回

1. 打开 **团队** → 选中团队 → 找到 **生产 SQL 审批**（或通知「生产审批待审」）。
2. 用状态筛选：**待审批** / 已执行 / 执行失败 / 已驳回。
3. 点开一条看 **审批详情**（连接 + SQL）。
4. 审阅后：
   - **批准并执行** → 由系统代为在该连接执行；成功或失败会回写状态
   - **驳回** → 可填驳回说明 → **确认驳回**
5. 将结果告知提交人（站内通知 / 飞书钉钉 Webhook，见 10.11）。

```text
成员写 SQL → 提交审批
    → 管理员审阅
    → 批准并执行（服务端代跑）或 驳回
    → 状态变为已执行 / 失败 / 已驳回
```

### 迁移 / 还原进生产

- **数据迁移向导**：目标为生产且需团队审批时，提交后 **不会立即执行**；管理员审批后再在向导侧执行（界面会提示「迁移计划已提交审批」）。
- **还原向导**：生产目标需勾选确认，并可能同样进入审批。

详见 [第 9 章](./09-schema-migration.md)、[第 5.6 节](./05-table-data.md)。

## 10.7 功能权限（账号权限）

**入口：** `Ctrl+,` → **账号权限**（需管理员）。

界面位置可参考设置侧栏分组（`06-settings-basic.png` 左侧「组织与安全」）。

### 怎么配

1. 选中账号或角色模板。
2. 按功能键勾选：工作台执行/保存/AI、资源树右键项、设置页入口、顶栏菜单等。
3. 保存后让用户重新登录或刷新会话（若界面有提示）。

### 常见策略模板（示例）

| 角色 | 放开 | 收紧 |
|------|------|------|
| 分析师 | 控制台只读执行、AI、导出 | DDL、还原、删连接 |
| 开发 | 开发连接读写、迁移预发 | 生产 DDL 需审批 |
| DBA | 漂移、DQ、迁移、健康 | 按需限制 AI 配额外操作 |

## 10.8 租户管理（multi 模式）

仅 **multi 租户** 部署显示。

**入口：** 设置 → **租户管理**。

| 能力 | 说明 |
|------|------|
| 开通组织 | 名称、Slug |
| 冻结 / 恢复 / 软删除 | 停用组织访问 |
| 成员与角色 | 如 `tenant_admin` |
| AI 日配额 | 组织级限额 |
| 用量导出 | 计费级能力可能仍在演进 |

单机模式无此菜单属正常。

## 10.9 审计

1. 在团队/平台审计入口查看操作记录。
2. 使用 **合规审计导出**（CSV / JSON）。
3. 网格内 DML 另有本地 Time-travel 审计（第 5 章）——与平台审计互补。

导出后可交给安全同学入库；与 SIEM 的深度对接以当前集成为准。

---

## 10.10 SSO（OIDC）与集成

1. 管理员打开 **设置 → 集成**。
2. 配置 OIDC（可参考仓库 `config/oidc.json.example`）：Authorization Code + PKCE。
3. 配置组 claim → 角色映射；注意 scopes 护栏。
4. 客户端可通过 `GET /api/auth/login-options` 发现登录方式。

当前不提供 SAML / LDAP / SCIM 组织树同步。

Webhook 可启用 HMAC 验签，用于外部系统接收事件。

---

## 10.11 通知通道

可推送：审批结果、定时失败、漂移、洞察 digest 等。

| 通道 | 说明 |
|------|------|
| Webhook | 可验签 |
| 飞书 / 钉钉 | 群机器人类配置 |
| HTTP 邮件网关 | 非原生 SMTP |

在 **设置 → 集成** 维护订阅事件。与审批相关的事件名示例：

- `prod_approval_pending`（生产审批待审）
- `prod_approval_decided`（生产审批已决）

成员用通知抽屉（`Ctrl+Shift+N`）查收站内副本。

---

## 10.12 共享连接与值班包（补充步骤）

1. 团队 → **共享设置**。
2. **共享连接**：填入连接 ID（逗号或换行）→ 保存。
3. 为每个共享连接设默认权限：**只读** / **读写（DML）** / **DDL**  
   （viewer 恒只读；admin/owner 恒 DDL。）
4. （可选）**值班连接包**：从已共享连接里勾选 → 仪表盘可快速定位。
5. （可选）**共享控制台**：填 `c-` 开头的已保存控制台 ID。
6. （可选）开启 **共享 SQL 历史**：成员可在侧栏 SQL 日志看到共享连接上的执行记录。
7. **共享查询 / AI 会话** Tab：把常用 SQL 或只读 AI 会话分享给团队检索打开。

## 10.13 管理员日常清单

- [ ] 生产连接均打上生产标签且共享范围最小
- [ ] 分析师无生产 DDL；危险 SQL 权限收敛
- [ ] **生产 SQL 审批** 有人值守；Webhook 已订 `prod_approval_*`
- [ ] 加入团队审批策略与邀请码管理得当
- [ ] AI 配额与打标表定期复审
- [ ] 审计导出周期归档
