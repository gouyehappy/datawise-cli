# 05 · 表与数据操作

本章覆盖打开表、编辑数据、导入导出、备份还原、ER 图、测试数据、AI 打标与脱敏导出。

---

## 5.1 打开表

| 方式 | 步骤 |
|------|------|
| 双击 | 资源树中双击 **表** 节点 |
| 右键 | **打开表** |
| 查看所有表 | 库节点 → **查看所有表** → 在清单中打开 |

进入后通常有多个子页：数据浏览、结构/属性、以及连接器提供的其他工具。

---

## 5.2 浏览与编辑数据

### 浏览

1. 打开表数据页，等待首屏加载。
2. 使用分页或滚动加载更多行。
3. 用列头排序、过滤（能力因数据源而异）。
4. 只读连接或无 `UPDATE` 权限时，单元格不可改。

### 编辑并保存

1. 单击单元格进入编辑，修改值。
2. 工具栏 **保存**（或提示的快捷键）。
3. 若失败：看约束（主键、外键、非空）、类型不匹配、权限错误。

### Time-travel（网格审计回滚）

针对**本客户端网格内**已记录的 DML 变更：

1. 打开变更历史。
2. 选择某一版本 → **一键回滚**。

说明：这是本地审计范围内的恢复，**不是**数据库 Flashback。生产误改仍应先有备份（5.6）。

---

## 5.3 测试数据生成

用于开发/测试库快速灌数。

1. 表 Tab 工具栏，或表右键，进入测试数据/生成入口。
2. 按列名启发式生成样例（如 `email`、`phone` 等）。
3. **预览** 将要执行的 SQL。
4. 选择 **执行** 写入，或 **导出 SQL** 稍后手工跑。

生产库不要随意执行；有审批策略时会拦截。

---

## 5.4 导入数据（CSV / Excel）

### 入口

表节点右键 → **导入数据**（或表 Tab 内导入）。

### 步骤

1. 选择文件（CSV / Excel，以界面支持为准）。
2. 设置分隔符、编码、是否含表头。
3. 做列映射（文件列 → 表列）。
4. 预览若干行。
5. 执行导入；成功会提示导入行数。

### 失败时

| 提示 | 含义 |
|------|------|
| 当前数据源不支持 CSV 导入 | 换支持的引擎或用 SQL 控制台 `LOAD`/`COPY` |
| 文件为空或格式无效 | 检查编码与分隔符 |
| 无法解析表所在连接 | 树上下文丢失，重新从树打开表 |

---

## 5.5 导出数据与导出 SQL

### 导出数据（结果/表内容）

表右键 → **导出数据**：按连接器导出为文件（常为 CSV）。空表会提示无可导出数据。

### 导出 SQL / 导出向导

库或表右键 → **导出 SQL** / **导出向导…**：

| 选项 | 生成内容 |
|------|----------|
| 仅结构 | `DROP IF EXISTS` + `CREATE TABLE` 等 |
| 结构 + 数据 | 追加 `INSERT` |
| 每表最大行数 | `0` = 不限制（仍受服务端上限） |
| 输出 | 下载 `.sql` 或复制到剪贴板 |

适合迁移脚本草稿；正式跨环境迁移请用第 9 章向导（带预检与进度）。

---

## 5.6 备份与还原

### 备份向导

入口：库/表右键 → **备份向导…**

1. 选择范围（单表或多表/整库）。
2. 选择仅结构或结构+数据，设置行数上限。
3. 下载 `.sql` 备份文件。
4. 生产环境会强调确认范围与保存位置。

### 还原向导

入口：库右键 → **还原向导…**

1. 确认 **目标** 连接与数据库（防止写错环境）。
2. **选择备份文件**。
3. 阅读预检摘要：语句数、写语句、DDL、是否含 `DROP`。
4. 生产环境勾选确认；可能进入审批流。
5. **开始还原**，等待完成。

**铁律：** 还原前再看一眼环境标签是不是「生产」。

## 5.7 元数据文档

库级右键 → **元数据文档**：汇总表/列注释等，用于交付或归档。缺少连接/库上下文时无法生成。

## 5.8 ER 图与正向建模

### 入口

数据库节点右键 → **ER 图**。

### 能做什么

1. 查看表与外键关系布局。
2. 在图上调整关系、改列（能力以当前版本为准）。
3. 批量 DDL：`ADD` / `DROP` / `MODIFY` / `RENAME` / `COMMENT` 等。
4. 生成 SQL → 审阅 → 执行（受权限与生产策略约束）。

大变更建议：ER 上设计 → Schema 对比确认差异 → 迁移向导执行（第 9 章）。

---

## 5.9 查看所有表与 AI 打标

### 为什么要打标？

AI 分析侧栏的「数据范围」**只显示已标记为 AI 可用的表**。不打标则无法把表纳入分析上下文（或只能走无范围的普通聊天）。

### 操作步骤

1. 库节点右键 → **查看所有表**。
2. 在清单中勾选/标记需要给 AI 用的表（如 `orders`、`users`）。
3. 打开 **AI 聊天 / AI 分析**（第 6 章）。
4. 在数据范围面板勾选这些表。
5. 再提问。

可随时取消标记，缩小模型可见面。

---

## 5.10 复制表、清空与删除

| 操作 | 入口 | 注意 |
|------|------|------|
| 复制表 | 右键 → 复制表 → 仅结构 / 结构+数据 | 确认目标库 |
| 清空表 | **清空表** | 不可撤销；生产慎用 |
| 删除表 | **删除表** | 二次确认；可能审批 |

---

## 5.11 脱敏导出

组织启用脱敏策略时，导出 CSV/文件会对敏感列打码。导出前确认：

- 列是否含 PII；
- 当前账号是否绕过脱敏（通常不应）；
- 合规是否允许下载到本地。

---

## 5.12 推荐场景对照

| 你想… | 用这个 |
|-------|--------|
| 快速看几行、改一两个值 | 打开表 + 网格编辑 |
| 复杂条件查询 | SQL 控制台 / VQB |
| 开发库灌数 | 测试数据生成 |
| 从 Excel 进表 | 导入数据 |
| 交出可执行脚本 | 导出 SQL |
| 防呆备份 | 备份向导 |
| 给 AI 用 | 查看所有表 → AI 打标 |
| 改表结构 | ER 图或 DDL + Schema 对比 |
