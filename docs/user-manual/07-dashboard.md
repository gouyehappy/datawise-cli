# 07 · 仪表盘

仪表盘是登录后的「首页」：看健康、进工作台、开 AI，并自定义关注的组件。

---

## 7.1 界面对照

![仪表盘](../assets/screenshots/01-dashboard.png)

**图中信息：**

| 区块 | 说明 | 你可以做什么 |
|------|------|--------------|
| 顶部按钮 | 进入数据库、打开 AI、拖拽排序、自定义组件 | 一键跳转到第 3 / 6 章场景 |
| 运行指标 | 连接数、SQL 次数、已存控制台、已启用插件 | 快速判断环境是否就绪 |
| 快捷操作 | 新建控制台、回到上次 Tab、AI 分析、管理插件 | 日常最高频入口 |
| 连接状态 | 各连接健康、检查连接、告警规则 | 跳转设置里的连接健康 |
| 右侧列 | 当前工作区、已保存控制台、已启用插件 | 恢复上下文、对照预设 |

---

## 7.2 日常怎么用（3 分钟）

1. 打开 **仪表盘**，看「连接状态」是否有红/黄告警。
2. 有告警 → 点 **检查连接** 或进入 **设置 → 连接健康** 调探测间隔。
3. 无告警 → **进入数据库** 开始查数，或 **AI 数据分析** 提问。
4. 需要写 SQL → **新建 SQL 控制台**。
5. 昨天的查询 → **回到上次 Tab** 或从「已保存控制台」打开。

---

## 7.3 自定义组件布局

1. 点 **自定义组件**（或仪表盘设置入口）。
2. 选择要显示的卡片（连接、活动、插件、团队相关等）。
3. 用 **拖拽排序** 调整列内顺序。
4. 返回仪表盘查看。

团队成员入队后，「值班连接包 / 最近分析」类组件更有价值（第 10 章）。

---

## 7.4 AI 推荐组件

1. 进入仪表盘设置 → **AI 生成组件**（需已配置 AI 模型）。
2. 用自然语言描述关注点，例如：「我想看生产连接健康和最近失败的定时任务」。
3. 确认推荐结果，落到指定列。
4. 不合适再手动删改组件。

---

## 7.5 只读分享（图表卡片）

适用于把某个图表快照发给同事、无需对方登录改数据：

1. 在支持分享的图表卡片上打开分享。
2. 创建链接，设置过期时间。
3. 将链接发给接收方（只读快照 MVP）。
4. 在 **管理分享链接** 中可随时撤销。

不要分享含敏感明细的卡片；过期与撤销要养成习惯。
